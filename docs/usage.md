## 1. 起動から停止まで

### 起動

Ollama サーバーとアプリの **2つのプロセスを起動** する必要があります。

```bash
# ターミナル 1: Ollama サーバーを起動（起動したままにする）
ollama serve

# ターミナル 2: アプリを起動
.venv\Scripts\activate
python main.py
```

> **Tip:** Ollama インストール後にシステムトレイに常駐している場合は `ollama serve` は不要です。

### 停止（終了方法）

起動後、Windows の **タスクトレイ**（画面右下の通知領域）にオレンジ色の「N」アイコンが表示されます。

```
終了方法（いずれか）:

  [方法1] タスクトレイのアイコンを右クリック → 「終了」
  [方法2] タスクトレイのアイコンをダブルクリック
  [方法3] 対象ウィンドウ（コメントを流しているアプリ）を閉じる
            ※ このときオーバーレイも自動で停止します
```

> **注意:** オーバーレイはタスクバーに表示されません。終了するには必ずトレイアイコンを使用してください。

## 3. コメント表示の仕組み

### 分析サイクル

```
[起動]
  │
  ▼
[スクリーンキャプチャ]  ← 8秒ごと（CAPTURE_INTERVAL_MS）
  │  PNG をローカル保持（ネットワーク送信なし）
  ▼
[Ollama (LLaVA) に送信]  ← localhost:11434
  │  画像 + プロンプトをローカルで処理
  ▼
[コメント文言を受信]  例: "すごい", "wwww", "なるほど" など
  │
  ▼
[オーバーレイに表示]  ← 300ms 間隔でコメントをずらして流す
```

### コメントの流れ方

- 画面を **10 行のレーン** に分割し、各レーンに 1 つずつコメントが流れます
- コメントはランダムな速さ・フォントサイズ・色で表示されます
- 黒いアウトライン付きで、どんな背景でも視認できます

## 4. 設定のカスタマイズ

`config.py` を直接編集して動作を調整します。

### タイミング設定

```python
CAPTURE_INTERVAL_MS = 8_000   # 分析間隔（ms）
                               # LLaVA の処理時間（GPU: 2〜5秒、CPU: 30秒以上）より
                               # 大きい値に設定することを推奨
COMMENT_SPAWN_DELAY = 300     # 同バッチのコメントをずらす間隔（ms）
```

> CPU 実行の場合は `CAPTURE_INTERVAL_MS = 60_000`（1分）程度に設定すると安定します。

### コメントの速さ

```python
COMMENT_SPEED_MIN = 4   # 最低速度（px/tick, 1tick ≈ 16ms）
COMMENT_SPEED_MAX = 8   # 最高速度
```

### レーン数・フォント

```python
LANE_COUNT    = 10    # 同時に流せる行数
FONT_SIZE_MIN = 22    # フォントサイズの最小値
FONT_SIZE_MAX = 36    # フォントサイズの最大値
```

### コメントの色

```python
COMMENT_COLORS = [
    (255, 255, 255, 255),  # 白
    (255, 255, 100, 255),  # 黄
    (100, 220, 255, 255),  # 水色
    (255, 130, 130, 255),  # ピンク
    (150, 255, 150, 255),  # 緑
]
```

RGBA タプル（0〜255）で自由に追加・変更できます。

### AI プロンプトの変更

```python
OLLAMA_PROMPT = (
    "..."  # ここを変更
)
```

## 5. モデルの選び方

| モデル | VRAM 目安 | 速度（GPU） | コメント品質 | ダウンロードサイズ |
|--------|----------|------------|------------|-----------------|
| `moondream` | 〜2GB | 速い（1〜3秒） | 普通 | 約 1.7GB |
| `llava` | 〜6GB | 普通（2〜5秒） | 良い | 約 4GB |
| `llava:13b` | 〜10GB | 遅め（5〜10秒） | 高い | 約 8GB |

### モデルの切り替え方法

```bash
# 使いたいモデルをダウンロード
ollama pull moondream
ollama pull llava:13b

# config.py を編集
# OLLAMA_MODEL = "moondream"  # または "llava:13b"
```

### インストール済みモデルの確認

```bash
ollama list
```

## 6. トラブルシューティング

### コメントが流れない・表示されない

**原因1: Ollama が起動していない**

```
[worker] Ollama に接続できません (http://localhost:11434)
```

ダイアログにこのエラーが表示された場合、別のターミナルで `ollama serve` を実行してください。

**原因2: モデルがダウンロードされていない**

```
[worker] モデル 'llava' が見つかりません。
```

```bash
ollama pull llava
```

**原因3: GPU メモリ不足**

コンソールに CUDA エラーが表示される場合は、より軽量なモデルに切り替えてください：

```python
# config.py
OLLAMA_MODEL = "moondream"
```

---

### 分析に時間がかかりすぎる（CPU 実行時）

CPU のみで LLaVA を実行すると 1 回の分析に 30 秒以上かかることがあります。

対処法:
1. `OLLAMA_MODEL = "moondream"` に変更（最も軽量）
2. `CAPTURE_INTERVAL_MS` を大きくする（例: `60_000`）

---

### オーバーレイがウィンドウに重ならない

対象ウィンドウが **最小化** されているか、**仮想デスクトップ** が異なる可能性があります。
ウィンドウを通常表示の状態で選択し直してください。

---

### クリックがオーバーレイに吸収される（背後のアプリを操作できない）

コンソールに以下のメッセージが出ているか確認してください：

```
[overlay] SetWindowLongW failed (error ...) — overlay may not be click-through
```

エラーが出ている場合、管理者権限でコマンドプロンプトを起動して `python main.py` を実行してください。

---

### コメントが詰まって重なる

`COMMENT_SPEED_MIN/MAX` を上げるか、`LANE_COUNT` を増やしてください。

---

### Ollama のポートを変更したい

別のポートで Ollama を起動している場合は `config.py` を変更してください：

```python
OLLAMA_HOST = "http://localhost:11434"  # ← 使用しているポートに変更
```

```bash
# 別ポートで Ollama を起動する場合
OLLAMA_HOST=0.0.0.0:12345 ollama serve
```
